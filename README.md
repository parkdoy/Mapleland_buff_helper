# 메이플랜드 버프 도우미 (Mapleland Buff Helper)

메이플랜드와 같은 게임에서 주기적으로 버프를 사용해야 할 때, 이를 자동으로 실행해주는 웹 기반 도우미 프로그램입니다.

## ✨ 주요 기능

- **웹 기반 UI**: 모든 설정을 웹 브라우저의 직관적인 인터페이스를 통해 할 수 있습니다.
- **다중 버프 지원**: 여러 종류의 버프를 각각 다른 주기로 등록하고 관리할 수 있습니다.
- **사용자 정의 설정**: 각 버프마다 원하는 키와 실행 주기(초 단위)를 자유롭게 설정할 수 있습니다.
- **시작 카운트다운**: '시작' 버튼을 누르면 3초의 카운트다운이 제공되어, 사용자가 게임 창을 활성화할 시간을 벌 수 있습니다.
- **겹침 방지 처리**: 여러 버프의 실행 시간이 겹칠 경우, 등록된 순서대로 0.4초의 간격을 두고 순차적으로 실행하여 오류를 방지합니다.

## 🚀 사용 방법 (일반 사용자용)

파이썬 설치 없이, `.exe` 파일로 프로그램을 실행하는 방법입니다.

1.  **프로그램 다운로드**
    - 이 깃허브 저장소의 **[Releases](https://github.com/사용자이름/저장소이름/releases)** 페이지로 이동합니다.
    - 최신 버전의 `app.exe` 파일을 다운로드합니다.

2.  **프로그램 실행**
    - 다운로드한 `app.exe` 파일을 더블클릭하여 실행합니다.
    - 잠시 후, 자동으로 기본 웹 브라우저가 열리면서 `http://127.0.0.1:5000` 주소로 프로그램 UI가 나타납니다.
      *(만약 브라우저가 자동으로 열리지 않으면, 직접 주소창에 `http://127.0.0.1:5000`을 입력해 주세요.)*

3.  **버프 설정**
    - `+ 버프 추가` 버튼을 눌러 원하는 만큼 버프를 추가합니다.
    - 각 행에 사용할 **키**와 **주기(초)**를 입력합니다.

4.  **시작 및 종료**
    - 게임 창을 활성화할 준비를 한 후, 웹 UI에서 `시작` 버튼을 누릅니다.
    - "3초 후 시작..." 카운트다운이 표시되는 동안, **게임 창을 클릭하여 활성화**합니다.
    - 3초가 지나면 설정된 첫 버프가 즉시 실행되고, 그 후부터 설정된 주기에 따라 자동으로 반복됩니다.
    - 자동 버프를 중지하려면 `중지` 버튼을 누릅니다.

## 🛠️ 사용 방법 (개발자용)

소스 코드를 직접 실행하거나 수정하고 싶은 경우, 아래 방법을 따릅니다.

1.  **저장소 클론**
    ```bash
    git clone https://github.com/사용자이름/저장소이름.git
    cd Mapleland_buff_helper
    ```

2.  **가상 환경 생성 및 활성화 (권장)**
    ```bash
    python -m venv venv
    .\venv\Scripts\activate
    ```

3.  **필요 라이브러리 설치**
    ```bash
    pip install -r requirements.txt
    ```

4.  **프로그램 실행**
    ```bash
    python app.py
    ```
    - 터미널에 나타난 안내에 따라 웹 브라우저에서 `http://127.0.0.1:5000` 주소로 접속합니다.

## 📦 빌드 방법 (.exe 파일 만들기)

직접 소스 코드를 수정하고 `.exe` 파일을 만들고 싶은 경우, 아래 방법을 따릅니다.

1.  **PyInstaller 설치**
    ```bash
    pip install pyinstaller
    ```

2.  **빌드 명령어 실행**
    - 프로젝트 루트 폴더(`app.py`가 있는 위치)에서 아래 명령어를 실행합니다.
    ```bash
    pyinstaller --onefile --windowed --add-data "templates;templates" app.py
    ```
    - `--onefile`: 모든 파일을 하나의 `.exe`로 묶습니다.
    - `--windowed`: 실행 시 불필요한 콘솔 창이 뜨지 않도록 합니다.
    - `--add-data "templates;templates"`: `index.html`이 포함된 `templates` 폴더를 빌드에 포함시킵니다.

3.  **결과 확인**
    - 빌드가 완료되면 `dist` 폴더가 생성되고, 그 안에 최종 결과물인 `app.exe` 파일이 만들어집니다.
